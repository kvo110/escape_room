Project Overview:
For this project, I will build a static HTML and CSS-based “Escape Room Challenge” website. The site will simulate an interactive escape room experience where users move through different puzzle rooms and unlock clues. Each room will include layered visual elements and CSS-only interaction techniques such as :target and :checked patterns to reveal hidden information.

The Adventure Pack will be used to create a sense of progression through checkpoints and visual cues that guide the user through the experience. The overall goal is to design a clean, immersive interface that demonstrates strong semantic HTML structure, reusable CSS components, responsive layouts, and meaningful animations.

The focus of this project will be on usability, visual consistency, accessibility, and thoughtful design decisions rather than overcomplicating features.


Proposed Wireframe:

Homepage (index.html)
- Immersive hero section with title and theme
- Short introduction to the escape challenge
- “Begin Challenge” call-to-action button
- Navigation bar (Home, Rooms, Clue Board, Final Unlock, Team)
- Visual checkpoint/timeline element (Adventure Pack feature)

Purpose:
The homepage introduces the concept of the escape room and clearly directs the user into the experience.


Puzzle Rooms Page (rooms.html)
- Responsive grid layout of room cards (Room 1, Room 2, Room 3)
- Each card includes:
  - Room title
  - Difficulty indicator
  - Unlock/progress status
- CSS-only clue reveal interactions

Purpose:
This page allows users to explore different puzzle rooms and interact with clues using CSS-based state changes.


Clue Board Page (clues.html)
- Organized archive of discovered clues
- Layered interface using :target for reveal behavior
- Visual indicators for solved vs. unsolved clues

Purpose:
The clue board reinforces structure and organization while allowing users to review progress in a clear and readable format.


Final Unlock Page (final.html)
- Progress indicator
- Final challenge message
- Unlock animation effect
- Celebration-style animation once unlocked

Purpose:
This page provides closure to the experience and highlights animation techniques used throughout the project.


Team Page (team.html)
- Project leader: Kenny Vo
- Time tracking log
- Development journal summary

Purpose:
This page fulfills submission requirements and documents the project development process.


Functionality:

Screen-to-Screen Transition Use Cases:
- Home to Rooms page through the main call-to-action button
- Rooms to Clue Board through navigation
- Rooms to Final Unlock once progression is complete
- Persistent navigation bar across all pages


Transition Implementation Details:
- CSS transitions for hover states on buttons and cards
- Navigation hover underline animation
- Smooth visual feedback for interactive elements


Transformation Implementation Details:
- transform: scale() for card hover emphasis
- transform: translateY() for subtle lift effects
- Small positional shifts when revealing clues


Animation Implementation Details:
At least three meaningful animations will be implemented:
1. Room card entrance animation (fade + slide effect)
2. Progress bar fill animation using keyframes
3. Unlock animation on the final challenge page

Animations will support user feedback and hierarchy rather than distract from content.


User Experience:

Proposed Use Cases:
- User arrives on homepage and reads instructions.
- User selects a room and interacts with puzzle elements.
- User checks the clue board to review progress.
- User reaches the final unlock page and completes the challenge.


Accessibility Notes:
- Proper semantic structure (header, nav, main, section, footer).
- Logical heading hierarchy (h1 to h2 to h3).
- Visible focus states for keyboard navigation.
- High-contrast color palette for readability.
- Use of prefers-reduced-motion media query where applicable.
- Descriptive alt text for relevant images.
