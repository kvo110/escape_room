What I built

A small escape-room website with a “neon lab evacuation” theme and a clear start-to-finish path.

  • Home: mission brief + route preview
  • Rooms: zone cards with CSS-only clue reveals
  • Clue Board: target-based clue browsing (no JavaScript)
  • Final Unlock: radio override + switch trigger + door animation
  • Team: project summary + testing notes

Key CSS-only interactions

These were the main “escape room” mechanics, implemented without JavaScript.

  • Reveal clue buttons: checkbox + adjacent sibling selectors
  • Clue Board navigation: :target highlights and show/hide
  • Final Unlock: radio selection enables the switch, then :has() unlocks the door
  • Accessibility: skip link, focus-visible rings, reduced-motion support

Testing notes

I checked the site on desktop and on mobile/tablet sizes to make sure the layout stays readable and the interactions still work without hover.

  • Verified navigation links on every page (no dead ends)
  • Verified every “Reveal clue” checkbox opens and closes correctly
  • Verified Clue Board :target behavior (selected clue shows, others hide)
  • Verified Final Unlock logic: wrong code shows denied, correct code enables the switch, switch opens door
  • Verified reduced motion: animations stop, but the UI is still usable
  • Verified mobile glow fix: background stays subtle on iPhone/iPad

Student note: If a browser doesn’t support :has(), the Final Unlock still displays the story content, but the door animation may not trigger. (Modern Chrome/Safari should support it.)
